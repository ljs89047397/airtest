## 4. DB 스키마 구조 (6 스키마)

| 스키마 | 용도 | 주요 테이블 수 |
|---|---|---|
| `com` | 공통 (사용자·기관·권한·결재·코드) | 20+ |
| `emp` | EMP (모니터링 계획) | 10 |
| `er` | ER + CEF + EUCR + OoM | 15+ |
| `vr` | VR (검증보고서) | 8 |
| `saf` | SAF (인증서·배치·공항·혼합) | 10 |
| `ptl` | 포털 (시뮬·CCR·감사로그) | 4 |
| **합계** | | **약 67개** |

명명 규칙:
- `tn_` : 트랜잭션 (수정 가능)
- `tc_` : 코드성 (마스터)
- `th_` : 이력
- `tn_emp_plan` → 도메인 + 엔티티 (snake_case)

# 보안 요구사항서 (Security Requirements)

> **시스템**: ICAS-CEMS | **작성일**: 2026-05-24 | **버전**: v1.0

## 1. 보안 위협 모델 (Threat Model)

| 위협 | 영향 | 대응 |
|---|---|---|
| 무차별 로그인 시도 (brute force) | 계정 탈취 | 5회 실패 → 자동 잠금 |
| 세션 하이재킹 | 사용자 사칭 | HttpOnly·Secure 쿠키 + 30분 만료 |
| CSRF | 의도하지 않은 요청 | XSRF-TOKEN 쿠키 + AJAX 헤더 자동 |
| SQL Injection | DB 탈취·변조 | MyBatis #{} 파라미터 바인딩 |
| XSS | 사용자 세션 탈취·악성 스크립트 | IcasEsc 출력 이스케이프 |
| 권한 우회 (IDOR) | 타사 데이터 접근 | DataScopeValidator + AuthorityInterceptor |
| 비밀번호 평문 노출 | 자격 증명 유출 | SHA-256 해시 저장, 로그 마스킹 |
| 감사 추적 누락 | 사후 조사 불가 | `ptl.th_user_actn` 모든 변경 적재 |
| 파일 업로드 | 악성 코드 업로드 | 화이트리스트 확장자 + 가상 디렉터리 |

## 2. 인증 (Authentication)

### 2.1 로그인 정책
| 항목 | 정책 | 구현 |
|---|---|---|
| 비밀번호 최소 길이 | 8자 | `password.jsp` 클라이언트 + Service 서버 검증 |
| 비밀번호 복잡도 | 영문 + 숫자 + 특수문자 | 정규식 (UI 실시간 표시) |
| 비밀번호 만료 | 90일 권장 (1차년도 미강제) | 2차년도 활성화 |
| 로그인 실패 잠금 | 5회 → 자동 잠금 | `AuthMapper.incrementPasswordFailCount` |
| 잠금 해제 | MOLIT/KOTSA 관리자만 | `POST /api/com/user/{userId}/unlock` |
| 비밀번호 해시 | SHA-256 + (운영 시 bcrypt 권장) | `kr.go.molit.icas.common.util.Sha256` |
| 로그인 이력 | 성공/실패 모두 적재 | `com.th_logn_hstry` |

### 2.2 세션 관리
| 항목 | 정책 | 구현 |
|---|---|---|
| 세션 만료 | idle 30분 | Tomcat web.xml session-timeout=30 |
| 만료 5분 전 경고 | 토스트 알림 | `header.jsp` setInterval 30s |
| 활동 시 자동 갱신 | click/keydown/scroll | debounce 60s + `/api/com/user/me` 호출 |
| 로그아웃 | POST /api/com/auth/logout + 세션 무효화 | `IcasLogoutHandler` |
| 쿠키 속성 | HttpOnly + (운영 시 Secure + SameSite=Strict) | Tomcat 9 기본 |

## 3. 권한 제어 (Authorization)

### 3.1 RBAC 구조
```
사용자 (tn_user)
    │
    └─ 1:N → 사용자-역할 매핑 (tn_user_role_mpng)
              │
              └─ 1:N → 역할-권한 매핑 (tn_sys_authrt_role_mpng)
                        │
                        └─ 1:N → 권한-프로그램 매핑 (tn_sys_authrt_prgrm_mpng)
                                  │
                                  └─ 1:1 → 프로그램 (tn_prgrm.api_path_prefix)
```

### 3.2 8 actor × 11박스 권한 매트릭스 (요약)
- **MOLIT_ADMIN/USER**: 전사 가시 + 승인 + 시스템 운영
- **KOTSA_REVIEWER/USER**: 전사 가시 + 검토·권고
- **AIRLINE_MANAGER/OPERATOR**: 본사 EMP·ER·CEF·EUCR·SAF만 가시·작성
- **VERIFIER_LEAD/MEMBER**: 본 검증기관 배정 운영사 VR 가시·작성

### 3.3 다층 권한 검증
1. **AuthorityInterceptor** — URL prefix 매칭 (1차 라우팅 차단)
2. **Service 레이어** — `DataScopeValidator.assertOprtrAccessible` (2차 데이터 스코프)
3. **DB 레이어** — `tn_*` 테이블의 `oprtr_id` 컬럼 자동 필터

## 4. CSRF 보호

| 항목 | 구현 |
|---|---|
| 토큰 발급 | Spring Security `CookieCsrfTokenRepository` (httpOnly=false) |
| 쿠키명 | `XSRF-TOKEN` |
| 헤더명 | `X-XSRF-TOKEN` |
| 자동 첨부 | `/resources/js/common/icas-csrf.js` 전역 `$.ajaxSetup` |
| 면제 경로 | `GET`, `HEAD`, `OPTIONS`, `TRACE` |
| 검증 위치 | `CsrfFilter` (Spring Security 필터 체인) |

## 5. 데이터 보호

| 항목 | 구현 |
|---|---|
| **비밀번호** | SHA-256 해시 (운영 단계 bcrypt 권장) |
| **개인정보 마스킹** | 사용자 휴대폰·이메일 부분 마스킹 (관리자 외) |
| **감사로그** | INSERT·UPDATE·DELETE 5W1H 적재 (`ptl.th_user_actn`) |
| **소프트 삭제** | `use_end_dt` 컬럼으로 논리 삭제, 5년 후 물리 삭제 |
| **트랜잭션** | `@Transactional` 단일화, 외부 송신 시 보상 트랜잭션 |
| **파일 업로드** | 화이트리스트 (pdf, xlsx, csv, jpg, png), 가상 디렉터리 |

## 6. 입력 검증 (Input Validation)

| 영역 | 도구 | 적용 |
|---|---|---|
| **SQL 파라미터** | MyBatis `#{}` 바인딩 | 모든 SQL 100% |
| **HTML 출력** | `IcasEsc.html(value)` 유틸 | 사용자 입력 출력 시 |
| **JSON 파싱** | Jackson + `@Valid` + Bean Validation | 모든 API 컨트롤러 |
| **파일명** | 한글 + 영문 + 숫자만, 경로 분리 차단 | FileUploadService |
| **날짜·숫자** | ISO 8601 / 정수·소수 정규식 | UI + 서버 양쪽 검증 |

## 7. 보안 점검 항목 (OWASP Top 10 매핑)

| OWASP | 항목 | 우리 시스템 |
|---|---|---|
| A01 | Broken Access Control | ✅ RBAC + DataScope |
| A02 | Cryptographic Failures | ✅ SHA-256 (bcrypt 권장) + HTTPS 권장 |
| A03 | Injection | ✅ MyBatis 바인딩 + IcasEsc |
| A04 | Insecure Design | ✅ 결재 라이프사이클 + 비즈룰 (VR 미해결 부적합 차단 등) |
| A05 | Security Misconfiguration | ⚠️ 운영 단계 Tomcat 보안 설정 강화 필요 |
| A06 | Vulnerable Components | ⚠️ 정기 의존성 점검 (OWASP Dependency-Check) |
| A07 | Identification & Auth | ✅ 5회 잠금 + 세션 만료 |
| A08 | Software & Data Integrity | ✅ CSRF + 트랜잭션 |
| A09 | Logging & Monitoring | ✅ 감사로그 + `/admin/health` |
| A10 | SSRF | ⚠️ 외부 API 콜 제한 (운영 단계 화이트리스트) |

## 8. 보안 사고 대응 (Incident Response)

| 단계 | 절차 |
|---|---|
| **탐지** | `/admin/health` + 감사로그 이상 패턴 모니터링 |
| **격리** | 의심 계정 즉시 잠금 (`POST /user/{id}/unlock` 반대) |
| **분석** | `ptl.th_user_actn` 조회 + `com.th_logn_hstry` 추적 |
| **복구** | DB 백업 시점 복원 (RPO ≤ 1h) |
| **사후** | 보안 패치 + 사용자 공지 + 운영 매뉴얼 갱신 |
